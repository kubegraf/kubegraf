Web Dashboard

Modern browser-based interface for comprehensive cluster management

Launching the Dashboard

# Default port (8080) kubegraf --web # Custom port kubegraf --web --port=3000

Open http://localhost:8080 (or your custom port) in your browser.

Dashboard Overview

The web dashboard provides:

  • Resource Overview - Quick summary of all resource types
  • Real-time Metrics - CPU, memory, and pod counts with sparklines
  • Resource Tables - Detailed view with action buttons
  • Connection Status - Live cluster connectivity indicator

Resource Management

Pods

View logs, exec shell, restart, delete, view YAML

Deployments

Scale replicas, restart, view YAML and details

Services

Port forward, view endpoints, YAML view

All Resources

StatefulSets, DaemonSets, Ingresses, ConfigMaps, CronJobs, Jobs, Nodes

Namespace Selection

Use the namespace dropdown in the header to:

  • Select a specific namespace to filter resources
  • Choose "All Namespaces" to view resources across the entire cluster

Port Forwarding

Port forwarding lets you access services running in your cluster locally:

  1. Navigate to the Services tab
  2. Click the PF (Port Forward) button on any service
  3. The service becomes accessible at localhost:[port]
  4. A green PF indicator appears at the bottom-right showing active forwards
  5. Click Stop to terminate the port forward
Note: Port forwards run in the background and persist until stopped or the server is restarted.

Topology View

Access the interactive topology visualization at /topology or click "Topology" in the navigation.

The D3.js force-directed graph shows:

  • Resource relationships (Deployments → ReplicaSets → Pods)
  • Service connections to pods
  • Ingress routing
  • Color-coded resource types

Interact by dragging nodes, zooming, and hovering for details.

Connection Status

The top-right corner shows your cluster connection status:

  • ● Connected - Cluster is reachable
  • ● Disconnected - Cluster is unreachable

When disconnected, a red error banner appears with troubleshooting information. The status updates automatically every 5 seconds.

WebSocket Updates

The dashboard uses WebSocket connections for real-time updates. Resources refresh automatically when changes occur in your cluster.

Graceful Degradation

The web dashboard starts even if your cluster is unreachable, allowing you to:

  • See connection error details
  • Wait for the cluster to become available
  • Troubleshoot connectivity issues
