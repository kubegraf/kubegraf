Solid.js UI

Modern reactive single-page application with advanced features

Overview

The Solid.js UI is a modern, feature-rich frontend for KubeGraf that provides a reactive, single-page application experience. Built with Solid.js for fine-grained reactivity, it offers superior performance and a polished user experience.

Getting Started

Prerequisites

  • Node.js 18+ and npm
  • KubeGraf backend running (kubegraf --web)

Installation

# Start the backend server kubegraf --web --port=8080 # In a new terminal, start the Solid.js frontend cd ui/solid npm install npm run dev

Open http://localhost:3000 in your browser.

Development: The Vite dev server proxies API requests to the backend at port 8080. In production, you can build the UI with npm run build and serve it statically.

Themes

The Solid.js UI includes 5 built-in themes that can be switched from the header:

Dark Light Midnight Cyberpunk Ocean

Themes are persisted in localStorage, so your preference is remembered across sessions.

Features

Dashboard

The dashboard provides an overview of your cluster with:

  • Cluster metrics (pods, deployments, services, nodes)
  • Resource status indicators
  • AI-powered insights and recommendations
  • Quick actions for common operations

Workloads

Manage all workload types with full CRUD operations:

Resource Operations
PodsShell, Logs, YAML, Describe, Restart, Delete
DeploymentsScale, Restart, YAML, Describe, Delete
StatefulSetsScale, YAML, Describe, Delete
DaemonSetsYAML, Describe, Delete
CronJobsTrigger, Suspend, YAML, Describe, Delete
JobsYAML, Describe, Delete

Network

  • Services: Port forwarding with session management, endpoint discovery
  • Ingresses: TLS status, host rules, backend services

Configuration

  • ConfigMaps: View and edit with syntax-highlighted YAML

Cluster

  • Nodes: Status, capacity, allocatable resources, conditions
  • Resource Map: Interactive topology visualization with D3.js

Security Dashboard

The Security section provides automated security analysis:

  • Security Score: 0-100 score based on best practices
  • Findings: Categorized by severity (Critical, High, Medium, Low)
  • Remediation: Actionable guidance for each finding

Security checks include:

  • Pods running as root
  • Missing SecurityContext
  • Privileged containers
  • Missing resource limits
  • Namespaces without NetworkPolicies
  • Ingresses without TLS
  • Services using insecure ports

Plugins

The Plugins section integrates with popular Kubernetes tools:

Helm

  • View installed Helm releases
  • Release status, chart version, app version
  • Upgrade, rollback, uninstall operations

ArgoCD

  • List ArgoCD applications
  • Sync status and health
  • Sync and refresh actions

Flux

  • View Flux GitRepositories, HelmReleases, Kustomizations
  • Reconciliation status
  • Force reconcile actions

Real-time Updates

The Solid.js UI uses WebSocket connections for real-time updates:

  • Live resource status changes
  • Event stream notifications
  • Connection status indicator

AI Chat

An AI assistant is available in the sidebar to help with:

  • Cluster troubleshooting
  • Kubernetes best practices
  • Resource configuration advice
  • Security recommendations

Technical Stack

  • Solid.js: Reactive UI framework with fine-grained reactivity
  • Vite: Fast build tool with HMR
  • Tailwind CSS: Utility-first CSS with CSS custom properties for theming
  • TypeScript: Type-safe development
  • D3.js: Topology visualization

Building for Production

# Build optimized bundle cd ui/solid npm run build # Preview the build npm run preview

The build output is in ui/solid/dist/ and can be served by any static file server.
