Configuration

Customize KubeGraf for your environment

Kubeconfig

KubeGraf uses the standard Kubernetes client configuration. It automatically reads from:

  1. $KUBECONFIG environment variable (if set)
  2. ~/.kube/config (default location)
  3. In-cluster config (when running inside Kubernetes)

Multiple Contexts

If your kubeconfig has multiple contexts, KubeGraf uses the current context. Switch contexts with:

kubectl config use-context my-cluster

Command Line Options

FlagDescriptionDefault
[namespace] Initial namespace to view default
--web Launch web dashboard instead of TUI false
--port=PORT Web server port (with --web) 8080
--version, -v Show version information -
--help, -h Show help message -

Examples

# Terminal UI in default namespace kubegraf # Terminal UI in production namespace kubegraf production # Web dashboard on default port kubegraf --web # Web dashboard on custom port kubegraf --web --port=3000 # Check version kubegraf --version

Metrics Server

For CPU and memory metrics, you need the Kubernetes Metrics Server installed in your cluster:

# Check if metrics server is installed kubectl get deployment metrics-server -n kube-system # Install metrics server (if not present) kubectl apply -f https://github.com/kubernetes-sigs/metrics-server/releases/latest/download/components.yaml

KubeGraf will still work without the metrics server, but resource usage data won't be available.

RBAC Requirements

KubeGraf needs read access to cluster resources. The minimum required permissions:

  • get, list, watch on pods, deployments, services, etc.
  • get on pod logs (for log viewing)
  • create on pods/exec (for shell access)
  • delete on resources (for delete operations)

Troubleshooting

Connection Issues

If KubeGraf can't connect to your cluster:

# Verify kubectl works kubectl cluster-info # Check current context kubectl config current-context # Test API access kubectl get namespaces

Metrics Not Showing

If CPU/memory metrics aren't appearing:

# Check metrics server kubectl top nodes kubectl top pods
